Data Exchange – The Role Of The Main And The Secondary Device

The template application implements a request-response type of communication. The Main device sends a request and waits for a response. The response is analyzed, and the process starts over. The secondary device waits for a request, and when it receives one, it begins analyzing and sends an appropriate response. At the end of this cycle, it goes back to waiting. In case of an error, it is counted and remembered. After a certain time from the occurrence of an error, the cycle restarts, and it goes back to the initial step.
